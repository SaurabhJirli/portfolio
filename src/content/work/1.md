## Overview

I've been working at **Touchcore Systems** in Pune for about 1.5 years. It's a smaller company, which means I've gotten to work across a lot of different things — from writing APIs to setting up Kafka pipelines to deploying services on Kubernetes.

### What I do day-to-day

- Write and maintain backend services in **Spring Boot** and **Java**
- Set up event-driven messaging between services using **Apache Kafka**
- Work with **PostgreSQL** for relational data, **MongoDB** for document storage, and **InfluxDB** for time-series metrics
- Build IoT data ingestion pipelines using **MQTT**
- Containerize services with **Docker** and deploy them via **Kubernetes**
- Write REST APIs and integrate with external services
- Participate in code reviews and help with system design discussions

### Things I've shipped

- Built a real-time IoT monitoring pipeline that ingests device data over MQTT into InfluxDB, with alerting when readings go out of range
- Set up a Kafka-based event pipeline that routes business events across multiple services, replacing a set of brittle synchronous API calls
- Helped containerize and migrate several services to Kubernetes, including setting up health checks and resource limits

## Tech I've used here

- **Backend:** Spring Boot, Java
- **Databases:** PostgreSQL, MongoDB, InfluxDB
- **Messaging:** Apache Kafka, MQTT
- **Infrastructure:** Docker, Kubernetes
- **Other:** REST APIs, Git, CI/CD pipelines

## Takeaway

Touchcore gave me the chance to work on real production systems early on. I've learned a lot about building things that actually need to stay up, handle real data, and work with other people's code.
