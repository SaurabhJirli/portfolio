I'm a backend engineer by day, so building a portfolio site with Next.js was partly a learning exercise. Here are some things I ran into and how I solved them.

## Why Next.js

I wanted something that could render static pages but also support dynamic routes for blog posts and projects. Next.js with the App Router does both. Server components mean most pages ship zero JavaScript to the browser.

## Markdown as a CMS

Instead of using a headless CMS, I went with plain markdown files in a `src/content/` folder. I use `gray-matter` for frontmatter parsing and `remark` to convert markdown to HTML. It's simple, version-controlled, and I can write posts in VS Code.

## Dark mode without the flash

The classic problem: your site loads in light mode for a split second before JavaScript kicks in and applies dark mode. The fix is a tiny blocking script in `<head>` that reads from localStorage and adds the `dark` class before the browser paints.

## Tailwind CSS v4

Tailwind v4 dropped the `tailwind.config.js` file in favor of CSS-based configuration with `@theme`. It took some getting used to, but the result is cleaner — theme tokens live in your CSS, not a JavaScript file.

## Things I'd do differently

- Set up MDX from the start instead of plain markdown — I keep wanting to embed components in posts
- Add reading time estimates earlier (I use `reading-time` now but added it late)
- Write more tests — the site works but I have zero automated checks

## The takeaway

Building a frontend project from scratch as a backend developer was humbling and educational. I have a lot more respect for frontend engineers now.
