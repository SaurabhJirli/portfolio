Before I started using Docker Compose, setting up my local dev environment was painful. Install Kafka? You need Zookeeper. Install PostgreSQL? Different version than production. Install InfluxDB? Good luck on Windows.

Now I have a single `docker-compose.yml` that brings up everything I need.

## My typical setup

```yaml
services:
  postgres:
    image: postgres:16
    ports:
      - "5432:5432"
    environment:
      POSTGRES_PASSWORD: localdev

  kafka:
    image: confluentinc/cp-kafka:7.5.0
    ports:
      - "9092:9092"

  influxdb:
    image: influxdb:2.7
    ports:
      - "8086:8086"
```

One `docker compose up` and I have a full environment. One `docker compose down` and it's gone.

## Why this works well

- **Consistent environments** — everyone on the team runs the same versions
- **No system pollution** — nothing installed on your machine besides Docker
- **Easy to reset** — blow away volumes and start fresh
- **Close to production** — same images, same configs

## Tips I've picked up

- Use `depends_on` with health checks so services start in order
- Mount volumes for data you want to persist between restarts
- Use `.env` files to keep passwords out of the compose file
- Add a `Makefile` with shortcuts like `make up`, `make down`, `make logs`

## The takeaway

If you're running `brew install` or downloading installers for every service your app depends on, try Compose instead. It's one of those tools where the setup cost is low and the payoff is immediate.
